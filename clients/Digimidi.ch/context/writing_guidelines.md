# Digimidi — Writing Guidelines 

## Voice & Tone
**Brand voice: The Knowledgeable Colleague** — an experienced vet who lives the daily realities of practice, not an outside vendor.

**Core attributes:** knowledgeable not lecturing · practical not promotional · empathetic not pitying · confident not arrogant · conversational not casual.

**Tone by content type:**
- *Problem-awareness* → understanding + validating.
- *Educational/how-to* → helpful + practical.
- *Industry trends* → informed + forward-looking.
- *Comparison/evaluation* → balanced + transparent.

## Writing Principles
1. **Start with the reader's reality** — open on a specific, relatable scenario, not company/product.
2. **Show, don't tell** — concrete numbers, before/after, real clinic examples.
3. **Respect veterinary expertise** — assume clinical knowledge; never condescend ("simply," "just").
4. **Lead with problems, not products** — establish the pain first; Digimidi appears as the solution.
5. **Write like you talk to a colleague** — natural, active voice, rhetorical questions, "we."

## Vocabulary
- **Use:** reclaim · modern vs. outdated · practice (not "business") · vets/veterinarians · pet families/clients · clinical work · gain visibility into · integrate/centralize.
- **Avoid:** synergy · leverage · revolutionary/game-changing/cutting-edge · disrupt · best/world-class · next-generation · robust · ecosystem · seamless · transform (sparingly).
- **Swiss specifics:** CHF (never $/€) · Romandie/Geneva/Lausanne references · multilingual reality (FR/DE/IT/EN) · nLPD · DD.MM.YYYY dates · British spelling · no US norms.
- **Vet terms:** proper terminology without over-explaining (DKA, OHE, etc.); "medical records," "appointment" (not "visit"), "client communication."

## Structure & Formatting
- **Opening (first 150 words):** hook scenario or surprising stat → validate problem → promise of payoff.
- **Body:** explore problem → why it happens → impact → solutions (general first) → Digimidi's approach → examples/numbers.
- **Closing:** recap insight → low-barrier next step (not always "book a demo") → leave value behind.
- **Formatting:** subheads every 250–400 words · paragraphs 2–4 sentences · bullets for 3+ items · bold sparingly (1–2/section) · avoid walls of text, ALL CAPS, clickbait · mobile-first.
- **Sentences:** avg 15–20 words, vary length, active voice.

## Key Do's & Don'ts
- **Do:** empathize, use specific numbers/scenarios, include Swiss context, back claims with evidence, add value even without conversion, write for scanners.
- **Don't:** minimize challenges, make unsupported claims, use fear-based manipulation, hide costs/tradeoffs, over-sell, use generic marketing language, be US-centric, patronize, create clickbait.

## SEO (when applicable)
- Primary keyword in H1 + first 100 words; semantic variations throughout; keyword in 50%+ of H2s; never keyword-stuff.
- **Meta description formula:** hook + specific benefit + credibility + CTA (~155 chars).
- Internal-link to 2–4 related articles with descriptive anchor text; build topic clusters.
- **Length:** problem-awareness 1,200–1,800 · how-to 1,500–2,500 · comparison 2,000–3,000 · trend 1,000–1,500. Quality over padding.

## Content Types
1. **Problem-awareness** — scenario → validation → impact → root cause → what's possible → solution → next step.
2. **Educational/how-to** — why it matters → context → steps → examples → mistakes → advanced tips.
3. **Comparison/decision-support** — the decision → what matters → evaluation framework → criteria → pitfalls → questions to ask.
4. **Industry trend/insight** — what's changing → context → current state → where it's going → implications → how to prepare.

## Final Principle: Always Add Value
Every article must pass: *"If someone reads this and never contacts Digimidi, did they still learn something useful?"* If no, rewrite. The goal is to be the generous veterinary colleague — that reputation makes Digimidi the obvious choice when a practice is ready to change.
